# 简单卡片

一个基于 `Vite + React + TypeScript` 的本地卡片生成器，用于快速制作适合公众号封面、小红书、微博、A4 纸张等场景的文字卡片，并直接导出为图片。

项目强调三件事：

- 模板切换快：内置 8 个卡片模板，覆盖简约、金句、书摘、便签、手写、代码等常见表达场景。
- 编辑反馈快：左侧选模板，中间实时预览，右侧调整比例、字体、排版、留白、配色和元素显隐。
- 导出流程短：支持 `PNG` 和 `JPG`，在浏览器内直接生成并下载。

## 功能概览

- 8 个内置模板
  - 默认简约
  - 金句语录
  - 书摘卡片
  - 便签笔记
  - iPhone 备忘录
  - 小红书大字报
  - 手写风格
  - 代码片段
- 支持多种画布比例
  - 自动跟随模板
  - `1:1`
  - `2.35:1` 公众号封面
  - `3:4` 小红书
  - `3:7` 小红书长图
  - `4:3` Instagram
  - `7:5` Pinterest
  - `9:16` 抖音
  - `12:16` 微博
  - `16:9` YouTube
  - `210:297` A4
  - `105:297` PDF 双列
- 支持实时编辑文本内容
- 支持字体、字号缩放、行距偏移、边距调整
- 支持背景色和文字色切换
- 支持图标、日期、标题、作者、二维码、页码、水印等元素显隐
- 支持一键重置为当前模板默认样式
- 支持导出为高清 `PNG` / `JPG`

## 适用场景

- 生成社交媒体图文封面
- 制作书摘、语录、便签、知识卡片
- 产出适合演示、打印或长图分发的文本型视觉内容
- 快速验证不同模板和版式的视觉效果

## 快速开始

### 环境要求

- Node.js 18+
- npm 9+

### 安装依赖

```bash
npm install
```

### 启动开发环境

```bash
npm run dev
```

默认会启动 Vite 开发服务器。终端会输出本地访问地址，通常是 `http://localhost:5173`。

### 生产构建

```bash
npm run build
```

构建产物位于 `dist/`。

### 本地预览构建结果

```bash
npm run preview
```

## 使用方式

1. 在左侧选择模板。
2. 在中间画布直接编辑标题或正文。
3. 在右侧切换比例、字体、排版、留白、配色或元素开关。
4. 点击右上角按钮，选择 `PNG` 或 `JPG` 导出。

## 核心设计

### 1. 模板驱动

每个模板统一定义以下信息：

- 基础布局尺寸
- 文本区域位置与默认内容
- 默认视觉样式
- 元素开关默认值

这让新增模板的成本较低，也便于在同一套编辑器里复用。

### 2. 全局状态集中管理

编辑器状态由 Zustand 管理，主要包括：

- 当前模板
- 当前文本内容
- 当前样式参数
- 当前画布比例
- 当前面板状态
- 当前元素显隐状态

这样模板切换、实时预览和导出可以共享同一份状态。

### 3. 浏览器内导出

导出使用 `html-to-image` 在前端完成，无需服务端参与。当前实现：

- 支持 `PNG` 和 `JPG`
- 默认使用 `pixelRatio: 2` 导出，兼顾清晰度与速度
- 导出文件名格式为 `card-{timestamp}.{ext}`

## 开发建议

- 新增模板时，优先复用 `src/templates/types.ts` 中的类型约束。
- 修改导出逻辑时，重点关注 `src/utils/export.ts`。
- 调整编辑器交互时，优先检查 `src/store/useStore.ts` 和 `src/components/Editor/StylePanel.tsx`。

## 已知特点

- 当前是纯前端项目，不依赖后端服务。
- 字体通过 Google Fonts 引入；在受限网络环境下，部分字体可能回退到系统字体。
- 文本编辑使用 `contentEditable`，适合轻量编辑场景，不是富文本编辑器。
